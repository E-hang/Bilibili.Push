SQLiteConnectionFlags 枚举

The extra behavioral flags that can be applied to a connection.

命名空间: System.Data.SQLite
程序集: Native.Tool.dll
语法
[Flags]
public enum SQLiteConnectionFlags : long

字段

名称 描述
AllowNestedTransactions

Allow transactions to be nested. The outermost transaction still controls whether or not any changes are ultimately committed or rolled back. All non-outermost transactions are implemented using the SAVEPOINT construct.
BindAllAsText

When binding parameter values, always bind them as though they were plain text (i.e. no numeric, date/time, or other conversions should be attempted).
BindAndGetAllAsInvariantText

When binding parameter values or returning column values, always treat them as though they were plain text (i.e. no numeric, date/time, or other conversions should be attempted) and always use the invariant culture when converting their values to strings.
BindAndGetAllAsText

When binding parameter values or returning column values, always treat them as though they were plain text (i.e. no numeric, date/time, or other conversions should be attempted).
BindDateTimeWithKind

When binding parameter values with the System.DateTime type, take their System.DateTimeKind into account as well as that of the associated SQLiteConnection.
BindDecimalAsText

When binding parameter values, always bind System.Decimal values as though they were plain text (i.e. not System.Decimal, which is the legacy behavior).
BindInvariantDecimal

When binding System.Decimal parameter values, always use the invariant culture when converting their values to strings.
BindInvariantText

When binding parameter values, always use the invariant culture when converting their values to strings.
BindUInt32AsInt64

When binding parameter values with the System.UInt32 type, use the interop method that accepts an System.Int64 value.
ConvertAndBindAndGetAllAsInvariantText

When binding parameter values or returning column values, always treat them as though they were plain text (i.e. no numeric, date/time, or other conversions should be attempted) and always use the invariant culture when converting their values to strings or from strings.
ConvertAndBindInvariantText

When binding parameter values, always use the invariant culture when converting their values to strings or from strings.
ConvertInvariantText

When binding parameter values, always use the invariant culture when converting their values from strings.
Default

The default extra flags for new connections.
DefaultAndLogAll

The default extra flags for new connections with all logging enabled.
DenyOnException

If an exception is caught when raising the Authorize event, the action should should be denied. If this is not specified, the action will be allowed instead.
DetectStringType

When returning column values, attempt to detect the type of string values by checking if they fully conform to those of the Null, Int64, Double, or DateTime types.
DetectTextAffinity

When returning column values, attempt to detect the affinity of textual values by checking if they fully conform to those of the Null, Int64, Double, or DateTime types.
GetAllAsText

When returning column values, always return them as though they were plain text (i.e. no numeric, date/time, or other conversions should be attempted).
GetDecimalAsText

When returning column values, always return System.Decimal values as though they were plain text (i.e. not System.Double, which is the legacy behavior).
GetInvariantDecimal

When returning System.Decimal column values, always use the invariant culture when converting their values from strings.
GetInvariantDouble

When returning System.Double column values, always use the invariant culture when converting their values from strings.
GetInvariantInt64

When returning System.Int64 column values, always use the invariant culture when converting their values from strings.
HidePassword

EXPERIMENTAL -- When opening a connection, attempt to hide the password from the connection string, etc. Given the memory architecture of the CLR, (and P/Invoke) this is not 100% reliable and should not be relied upon for security critical uses or applications.
InterruptOnException

If an exception is caught when raising the Progress event, the operation should be interrupted. If this is not specified, the operation will simply continue.
LogAll

Enable all logging.
LogBackup

Enable logging of backup API errors.
LogBind

Enable logging of all bound parameter strongly typed values.
LogCallbackException

Enable logging of all exceptions caught from user-provided managed code called from native code via delegates.
LogDefault

The default logging related flags for new connections.
LogPreBind

Enable logging of all bound parameter types and raw values.
LogPrepare

Enable logging of all SQL statements to be prepared.
MapIsolationLevels

Enable mapping of unsupported transaction isolation levels to the closest supported transaction isolation level.
NoBindFunctions

Skip binding any functions provided by other managed assemblies when opening the connection.
NoConnectionPool

Disable using the connection pool by default. If the "Pooling" connection string property is specified, its value will override this flag. The precise outcome of combining this flag with the UseConnectionPool flag is unspecified; however, one of the flags will be in effect.
NoConvertSettings

Skip querying runtime configuration settings for use by the SQLiteConvert class, including the default System.Data.DbType value and default database type name. NOTE: If the DefaultDbType and/or DefaultTypeName properties are not set explicitly nor set via their connection string properties and repeated calls to determine these runtime configuration settings are seen to be a problem, this flag should be set.
NoExtensionFunctions

Skip adding the extension functions provided by the native interop assembly.
NoGlobalTypes

Disable using global mappings between type names and System.Data.DbType values. This may be useful in some very narrow cases; however, if there are no per-connection type mappings, the fallback defaults will be used for both type names and their associated System.Data.DbType values. Therefore, use of this flag is not recommended.
NoLoadExtension

Prevent this SQLiteConnection object instance from loading extensions.
None

No extra flags.
NoVerifyTextAffinity

When returning column values as a System.String, skip verifying their affinity.
NoVerifyTypeAffinity

When returning column values, skip verifying their affinity.
RollbackOnException

If an exception is caught when raising the Commit event, the transaction should be rolled back. If this is not specified, the transaction will continue the commit process instead.
StickyHasRows

When the HasRows property is used, it should return non-zero if there were ever any rows in the associated result sets.
StrictConformance

EXPERIMENTAL -- Enable strict conformance to the ADO.NET standard, e.g. use of thrown exceptions to indicate common error conditions.
StrictEnlistment

Enable "strict" transaction enlistment semantics. Setting this flag will cause an exception to be thrown if an attempt is made to enlist in a transaction with an unavailable or unsupported isolation level. In the future, more extensive checks may be enabled by this flag as well.
TraceWarning

Enable tracing of potentially important [non-fatal] error conditions that cannot be easily reported through other means.
UnbindFunctionsOnClose

Attempt to unbind all functions provided by other managed assemblies when closing the connection.
UseConnectionAllValueCallbacks

Enables use of all per-connection value handling callbacks.
UseConnectionBindValueCallbacks

Enable using per-connection mappings between type names and SQLiteBindValueCallback values. Also see the ClearTypeCallbacks(), TryGetTypeCallbacks(String, out SQLiteTypeCallbacks), and SetTypeCallbacks(String, SQLiteTypeCallbacks) methods.
UseConnectionPool

Enable using the connection pool by default. If the "Pooling" connection string property is specified, its value will override this flag. The precise outcome of combining this flag with the NoConnectionPool flag is unspecified; however, one of the flags will be in effect.
UseConnectionReadValueCallbacks

Enable using per-connection mappings between type names and SQLiteReadValueCallback values. Also see the ClearTypeCallbacks(), TryGetTypeCallbacks(String, out SQLiteTypeCallbacks), and SetTypeCallbacks(String, SQLiteTypeCallbacks) methods.
UseConnectionTypes

Enable using per-connection mappings between type names and System.Data.DbType values. Also see the ClearTypeMappings(), GetTypeMappings(), and AddTypeMapping(String, DbType, Boolean) methods. These per-connection mappings, when present, override the corresponding global mappings.
UseParameterAnythingForTypeName

Enables use of all applicable SQLiteParameter properties as fallbacks for the database type name.
UseParameterDbTypeForTypeName

If the database type name has not been explicitly set for the parameter specified, fallback to using the database type name associated with the System.Data.DbType value.
UseParameterNameForTypeName

If the database type name has not been explicitly set for the parameter specified, fallback to using the parameter name.
WaitForEnlistmentReset

EXPERIMENTAL -- Enable waiting for the enlistment to be reset prior to attempting to create a new enlistment. This may be necessary due to the semantics used by distributed transactions, which complete asynchronously.
